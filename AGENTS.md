### 1. README 文档更新协议
每次执行任务生成新增内容或修改内容后，必须同步更新项目根目录下的 `readme.md` 文件。

**绝对约束：**
- `[禁止]` 修改或删除文件中与本次变更无关的任何原有内容，必须保持原样。
- 仅允许在文件中增加或修改与当前任务直接相关的描述。

**版本信息写入逻辑：**
- **定位当前版本**：在 `readme.md` 中找到当前版本号对应的区块。
- **分支 A（无当前版本信息）**：若原文件中尚未包含当前版本的改动信息，必须在**上一个版本的上方**新增一行，写入当前的新增内容。
- **分支 B（已有当前版本信息）**：若原文件中已存在当前版本的改动信息，必须在当前版本信息的**末尾追加**本次新增内容。

### 2. Git 版本控制协议
文档更新完成后，必须执行以下 Git 操作：

- **本地同步**：自动执行 `git add` 和 `git commit`，将变更提交到本地仓库。
- `[禁止]` 远程推送：`[禁止]` 执行 `git push` 或任何向远程仓库推送的操作。
- **推送触发条件**：远程仓库的推送操作**必须**等待用户明确要求（如“推送到远程”、“push”）后方可执行。
